# Информационная система библиотеки

Проект по разработке информационной системы для библиотеки с поддержкой проката книг, учета читателей, штрафов, скидок, финансовых показателей и аналитических запросов.

## О проекте

Данный проект представляет собой информационную систему для библиотеки, в которой реализован не только классический учет книг и читателей, но и коммерческий прокат изданий. Система позволяет вести каталог книг, регистрировать читателей, оформлять выдачи и возвраты, учитывать просрочки, рассчитывать стоимость аренды, штрафы и скидки, а также формировать аналитические выборки и отчеты.

В основе проекта лежит база данных на **MySQL**, дополненная **SQL-запросами**, **хранимыми процедурами**, **триггерами** и пользовательским интерфейсом на **Python**.

## Основные возможности

- учет читателей библиотеки;
- учет книг, авторов, жанров, издательств и экземпляров;
- регистрация выдачи и возврата книг;
- отслеживание просроченных выдач;
- расчет стоимости аренды;
- учет залоговой стоимости книг;
- применение скидок для отдельных категорий читателей;
- начисление штрафов за просрочку или повреждение книги;
- учет поступлений и списаний экземпляров;
- аналитические SQL-запросы по работе библиотеки;
- использование хранимых процедур для типовых операций;
- применение триггеров для проверки данных и поддержки целостности;
- разграничение прав доступа для разных пользователей системы;
- визуализация данных и работа с отчетностью.

## Используемые технологии

- **MySQL** — система управления базой данных;
- **SQL** — создание схемы, запросы, процедуры, триггеры, роли и права доступа;
- **Python** — интеграция с базой данных, пользовательский интерфейс и обработка данных.

## Цель проекта

Цель проекта — разработка полноценной информационной системы для библиотеки с возможностью коммерческого проката книг. В рамках проекта были выполнены:

- анализ предметной области;
- выделение сущностей, атрибутов и связей;
- построение ER-диаграммы;
- нормализация базы данных;
- реализация реляционной схемы в MySQL;
- написание SQL-запросов;
- разработка хранимых процедур и триггеров;
- проектирование пользовательского интерфейса;
- обработка и визуализация данных.

## Описание предметной области

Система разработана для библиотеки города Демидова. В отличие от обычной библиотеки, здесь часть книг выдается не только для чтения, но и в коммерческий прокат. Для таких книг учитываются:

- название произведения;
- авторы;
- жанры;
- издательство;
- залоговая стоимость;
- стоимость аренды за день.

Система также хранит информацию о читателях, их роде деятельности и специализации, чтобы можно было учитывать скидки для разных категорий пользователей. Для каждой выдачи фиксируются дата выдачи, ожидаемая дата возврата, фактическая дата возврата, стоимость аренды, штрафы и признак повреждения книги.

## Структура базы данных

База данных спроектирована с использованием трехуровневого подхода:

- инфологическая модель;
- логическая модель;
- физическая модель.

В базе данных реализовано **16 сущностей**, включая основные таблицы и таблицы связей.

### Основные сущности

- `readers` — читатели;
- `occupations` — род деятельности читателей;
- `discounts` — скидки;
- `discount_occupations` — связь скидок и рода деятельности;
- `rentals` — выдачи книг;
- `inventory_logs` — история поступлений и списаний;
- `librarians` — сотрудники библиотеки;
- `books` — книги;
- `authors` — авторы;
- `publishers` — издательства;
- `book_copies` — экземпляры книг;
- `genres` — жанры;
- `book_genre` — связь книг и жанров;
- `book_author` — связь книг и авторов;
- `specializations` — специализации читателей;
- `specialization_occupation` — связь специализации и рода деятельности.

## Нормализация

Структура базы данных приведена к **третьей нормальной форме (3НФ)**. Это позволяет:

- уменьшить избыточность данных;
- избежать аномалий вставки, удаления и обновления;
- обеспечить корректную организацию связей между сущностями;
- повысить надежность и целостность данных.

## Реализованные SQL-запросы

В проекте предусмотрены запросы для решения типовых задач библиотеки, например:

- получение списка читателей по специализации и роду деятельности;
- поиск читателей, у которых на руках находится определенная книга;
- получение списка читателей, которые брали указанную книгу за определенный период;
- расчет выработки библиотекарей;
- поиск просроченных выдач;
- получение данных о поступлении и списании литературы;
- расчет среднего количества выдач у библиотекаря;
- расчет среднего времени возврата книг по жанру;
- расчет финансовых показателей по конкретной книге;
- получение топ-5 самых популярных книг по числу выдач.

## Хранимые процедуры

Все основные запросы в системе реализованы через **хранимые процедуры**. Это позволяет:

- упростить повторное использование SQL-логики;
- передавать входные параметры;
- централизовать бизнес-логику на уровне базы данных;
- выполнять проверки корректности входных данных;
- уменьшить количество ошибок при работе пользователей с системой.

## Триггеры

Для автоматизации работы с данными в базе реализованы **триггеры**. Они используются для:

- проверки корректности дат выдачи и возврата;
- предотвращения некорректного ввода данных;
- автоматического контроля бизнес-правил;
- поддержания целостности данных при вставке и обновлении записей.

## Пользователи и роли

В системе предусмотрены разные роли пользователей с отдельными правами доступа.

### Директор библиотеки

- просмотр всех таблиц базы данных;
- обновление данных сотрудников;
- управление скидками.

### Библиотекарь

- работа с читателями;
- оформление выдач и возвратов;
- просмотр просроченных книг;
- работа с характеристиками читателей.

### Библиограф

- добавление и обновление информации о книгах;
- работа с авторами, издательствами и жанрами;
- учет поступлений и списаний.

### Администратор БД

- полный доступ ко всем таблицам;
- администрирование структуры базы данных.

## Отчеты и аналитика

Система позволяет формировать выходные документы и аналитические данные, включая:

- финансовый отчет по выручке и штрафам;
- список просроченных выдач;
- отчет по книжному фонду;
- статистику по востребованности книг;
- показатели по работе сотрудников библиотеки.

## Пользовательский интерфейс

Проект включает пользовательский интерфейс на Python для работы сотрудников библиотеки с базой данных. Интерфейс предназначен для выполнения повседневных операций:

- регистрация читателей;
- поиск информации по книгам и читателям;
- оформление выдачи и возврата;
- выполнение аналитических процедур;
- просмотр результатов запросов и отчетов.

## Как запустить проект

1. Создать базу данных в MySQL.
2. Импортировать SQL-скрипты со схемой и тестовыми данными.

5. Настроить параметры подключения к базе данных в Python-приложении.
6. Запустить приложение.
